# LLM ルーティングルール (NIM優先・トークン節約モード)

## 概要
Geminiのコンテキストウィンドウを節約し、クレジットを効率的に使用するため、**「重い処理」や「定型的な大幅な生成」は積極的に NVIDIA NIM（Qwen3.5）へ外部委譲**する。

## タスク振り分け方針（積極委譲）

### 1. NVIDIA NIMに「即」委譲すべきタスク（トークン消費大）

| タスクカテゴリ | 具体例 | 委譲のメリット |
| :--- | :--- | :--- |
| **新規ファイル生成** | Webアプリの雛形(HTML/CSS/JS)、スクリプト一式 | 数千トークンの出力をNIMに任せる |
| **大規模リファクタリング** | 100行以上の既存コードの構造変更、関数分割 | 文脈理解と生成の両方を外に出す |
| **デザイン・CSS全般** | プレミアムなUIデザイン、複雑なアニメーション | 試行回数が多くなるUI調整をNIMで回す |
| **テスト・ドキュメント作成** | 多数のテストケース生成、JSDoc/READMEの大量生成 | 副次的なドキュメント作業を外注する |
| **データ変換・スクレイピング** | 大量データの正規化、複雑な正規表現パターン作成 | 試行錯誤が必要なロジックをNIMで行う |

### 2. 自分（Gemini）が処理すべきタスク（最小限に維持）
- ルーティングの決定、ファイル操作のオーケストレーション
- 回答の最終確認とデバッグ
- 極めて重要なアーキテクチャ設計（全体方針）

## 委譲を加速させるためのプロンプト例
私がNIMに依頼する際、以下のフラグを付与して「一発回答」を促し、往復トークンを減らします。
- `回答はファイルの内容のみを出力してください`
- `ステップバイステップの解説は不要です。コードのみ出力してください`

## 使用モデル
- NIM デフォルト: `qwen/qwen3.5-122b-a10b`

> [!TIP]
> Geminiに「この処理をNIMでやって」と明示的に指示すると、Geminiは思考をスキップして直接 `llm_task.sh` を呼び出します。
